What is the Capstone?

  • A end-to-end data science project that integrates all course skills
  • You choose a real-world problem, collect data, analyse it, build a model, and present findings
  • Designed to demonstrate industry-relevant competency — not just technical exercises
  • Completed over 2 weeks (part-time), delivered as a 5-minute video presentation
  • Three structured briefs are provided — or propose your own
Data Science & Analytics | Skills Union

Capstone Goals

  • Problem framing — translate a real-world question into a data science task
  • Reproducibility — others can run your code end-to-end without hand-holding
  • Communication — present findings clearly to a non-technical audience
  • Full pipeline — data collection → cleaning → EDA → modelling → insights
  • Demonstrate both technical competency and business acumen

Project Requirements

Five core areas for all options

Data Science & AI | Skills Union × Localized

1. Data Collection & Preparation

  • Source and document your dataset(s) — record origin URL and access date
  • Identify and resolve data quality issues (missing values, duplicates, outliers)
  • Apply appropriate preprocessing techniques (normalisation, encoding, etc.)
  • Split data correctly — no data leakage between train and test sets
  • Store raw and processed data in clearly labelled folders
Data Science & Analytics | Skills Union

2. Exploratory Data Analysis

  • Conduct statistical summaries — distributions, correlations, skew
  • Create at least 3 meaningful visualisations with clear titles and labels
  • Extract and articulate key insights — what patterns surprised you?
  • Explore relationships between features before modelling
  • Document findings with markdown cells — notebooks should read like a report
Data Science & Analytics | Skills Union

3. Modelling & Analysis

  • Apply at least one appropriate ML technique (regression, classification, clustering)
  • Start with a baseline model before adding complexity
  • Evaluate performance using metrics appropriate to the problem (e.g., RMSE vs. F1)
  • Use cross-validation or a held-out test set — not just training accuracy
  • Document your model selection rationale — why this algorithm?
Data Science & Analytics | Skills Union

4. Results & Insights

  • Present clear findings — what does the data tell us about the problem?
  • Support every conclusion with evidence from the data or model output
  • Discuss practical implications — what actions could a stakeholder take?
  • Acknowledge limitations — what can your model not do or predict reliably?
  • Suggest future improvements — more data, different features, better models
Data Science & Analytics | Skills Union

5. Technical Implementation

  • Use industry-standard tools: Python, pandas, scikit-learn, matplotlib / plotly
  • Organise code across numbered notebooks (1-eda, 2-preprocessing, 3-modeling)
  • Follow PEP 8 style and include comments explaining non-obvious steps
  • Include a requirements.txt so others can reproduce your environment
  • Test that your notebooks run top-to-bottom without errors in a clean environment

Assessment & Common Pitfalls

What markers look for — and what to avoid

Data Science & AI | Skills Union × Localized

Assessment Focus Areas

  • Data quality — are issues identified and handled appropriately?
  • EDA depth — do visualisations reveal genuine insight, not just describe data?
  • Modelling rigour — is there a baseline? Is evaluation methodology sound?
  • Communication — is the story clear to a non-technical audience?
  • Reproducibility — can a marker clone and run the repo without modifications?
  • Documentation — are decisions explained with reasoning, not just code?
Data Science & Analytics | Skills Union

Common Pitfalls: Data & Modelling

  • Not exploring data before modelling — always do EDA first
  • Ignoring missing values or outliers — document what you found and what you did
  • Data leakage — never let test-set information influence training
  • No baseline model — always start simple before going complex
  • Overfitting — validate on held-out data, not just training accuracy
  • Wrong metrics — accuracy is misleading on imbalanced datasets; choose carefully
Data Science & Analytics | Skills Union

Common Pitfalls: Presentation & Docs

  • Exceeding the 5-minute limit — practice and time yourself beforehand
  • Too much technical jargon — explain for a general business audience
  • Poor visual design — clear labels, legible fonts, high contrast
  • No coherent story — connect problem → data → findings → recommendation
  • Insufficient code comments — explain the *why*, not just the *what*
  • No discussion of limitations — acknowledging weaknesses shows critical thinking

Submission Process

GitHub repository + video presentation

Data Science & AI | Skills Union × Localized

GitHub Repository Requirements

  • Create a public repository named capstone-project (or similar)
  • Folder structure: data/, notebooks/ (1-eda, 2-preprocessing, 3-modeling), src/, docs/
  • README.md must include: problem statement, dataset description, methodology, key findings, setup instructions
  • Include requirements.txt listing all Python dependencies
  • All notebooks must be committed with outputs visible (run before pushing)
Data Science & Analytics | Skills Union

Video Submission

  • Record using OBS Studio, Loom, or any screen-recording tool
  • Upload to YouTube, Vimeo, or Google Drive — unlisted or public
  • Ensure the link is accessible without login before submitting
  • Submit both the repository URL and video URL via the course platform
  • Recommended: use Canva or slides to support your narrative visually

Final Submission Checklist

  • Repository is public and all code is committed
  • Notebooks run top-to-bottom without errors from a fresh clone
  • requirements.txt is present and complete
  • README.md is clear and complete (problem, data, method, findings, setup)
  • Video is accessible, within 5 minutes, and clearly structured
  • Both URLs submitted via the course platform before the deadline
