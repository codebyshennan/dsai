What is Data Storytelling?

Why narrative matters in analytics

Data Science & Analytics | Skills Union

Data Alone is Not Enough

  • Raw data = raw ingredients (flour, sugar, eggs) — hard to digest on their own
  • Narrative = the recipe — provides structure, flow, and meaning
  • Story = the finished dish — engaging, memorable, and drives action
  • Studies show people remember stories 22× more than facts alone
  • Traditional reports: ~40% comprehension | Visualisation: ~65% | Data storytelling: ~85%
  • Goal: transform numbers into insights that any audience can understand and act on
Data Science & Analytics | Skills Union

The Three Pillars of Data Storytelling

  • Data: accurate, relevant, well-sourced evidence that supports your message
  • Narrative: a logical flow that guides the audience from context to insight to action
  • Visuals: charts and diagrams that make patterns and relationships instantly visible
  • Remove any pillar and the story weakens: data without narrative overwhelms; narrative without data is opinion; visuals without data misleads
  • The strongest presentations combine all three deliberately

Visual Storytelling Principles

One message per chart; hierarchy of emphasis

Data Science & Analytics | Skills Union

One Message Per Visualisation

  • Every chart should answer exactly one question — if it answers two, split it
  • The chart title should state the insight, not just describe the data (e.g. 'Revenue grew 30% after the product launch', not 'Revenue over time')
  • Remove everything that does not support the one message: gridlines, borders, redundant legends
  • Adequate white space helps the eye focus on the signal
  • Cognitive load is real — simpler charts get read; cluttered ones get skipped
Data Science & Analytics | Skills Union

Visual Hierarchy: Guide the Eye

  • Primary elements (the insight): bold, bright, large — your key metric or finding
  • Secondary elements (supporting data): medium weight and size — context that explains the insight
  • Background elements (reference): light, neutral — axes, benchmarks, grid
  • Use size, colour, and position to signal importance — not decoration
  • The eye naturally moves top-left to bottom-right (Z-pattern) and down the left edge (F-pattern)
  • Place the most important number or chart where the eye lands first

Choosing Visuals for Your Story

Match the chart type to the narrative goal

Data Science & Analytics | Skills Union

Chart-to-Narrative Matching

  • Comparison across categories → bar chart (horizontal when labels are long)
  • Trend over time → line chart (smooth for trends; sharp for actual values)
  • Distribution and spread → histogram or box plot
  • Outlier analysis → box plot (shows quartiles and whiskers)
  • Correlation between two variables → scatter plot (add trend line for clarity)
  • Parts of a whole → pie or donut chart (limit to 5–7 segments)
  • Flow or journey → Sankey diagram or funnel chart
Data Science & Analytics | Skills Union

Common Chart Mistakes to Avoid

  • Using a pie chart for time-series data — use a line chart instead
  • Too many categories in one chart — group small slices into 'Other'
  • Dual y-axes — almost always misleading; split into two separate charts
  • 3D charts — distort proportions; never use decorative 3D
  • Truncated y-axis starting above zero — exaggerates differences; start at zero for bar charts
  • Missing labels and units — always label axes and state the unit

Colour and Annotation for Emphasis

Guide the eye to what matters

Data Science & Analytics | Skills Union

Using Colour Purposefully

  • Sequential palette: for ordered data (light → dark = low → high values)
  • Categorical palette: for distinct groups that need clear differentiation
  • Diverging palette: for data that spans positive and negative (e.g. profit/loss)
  • Use one highlight colour to draw attention to the key data point — grey out everything else
  • Semantic colour: red = bad/alert, green = good/safe, blue = neutral — be consistent throughout
  • Always test for colour-blind accessibility — avoid red/green only distinctions
Data Science & Analytics | Skills Union

Annotations: Direct Labels over Legends

  • Direct labels on lines and bars remove the need for legends (which force eye movement)
  • Callout annotations highlight the single most important data point with a short phrase
  • Reference lines (averages, targets, benchmarks) add context without clutter
  • Arrows and circles draw attention to anomalies, peaks, or turning points
  • Keep annotation text short — one phrase, not a sentence
  • The annotation should say what the data point means, not just what it is

Narrative Frameworks

Structuring your message for maximum impact

Data Science & Analytics | Skills Union

The SCR Framework

  • Situation: the current state and context — what is happening right now?
  • Complication: the challenge, problem, or change — what makes this interesting or urgent?
  • Resolution: the insight and recommended action — what should we do about it?
  • SCR is concise and works well for business stakeholders with limited time
  • Example: 'Our churn is 5% (Situation). Half of churned users left in the first 7 days (Complication). We need an improved onboarding flow targeting the first week (Resolution).'
  • Every data presentation can be reduced to one SCR statement — write it before building slides
Data Science & Analytics | Skills Union

The Classic Narrative Arc

  • Hook: the attention-grabbing opening — a surprising fact, bold question, or stakes statement
  • Setup: context and background that the audience needs to understand the story
  • Journey: guided discovery through data — build understanding one 'aha' moment at a time
  • Reveal: the key insight — the main finding that changes how the audience thinks
  • Call to action: specific, measurable next steps — who does what by when
  • Use the arc for longer presentations; use SCR for executive summaries and dashboards
Data Science & Analytics | Skills Union

Building the Narrative Arc

Hook → Context → Conflict → Resolution

Data Science & Analytics | Skills Union

Crafting a Strong Hook

  • Question hook: 'What if we could predict churn before it happens?' — creates curiosity
  • Surprise hook: 'Our biggest competitor is actually our own website' — challenges assumptions
  • Stakes hook: 'Every minute we wait costs us another 100 customers' — creates urgency
  • The hook sets the frame for everything that follows — spend time on it
  • Weak hook: 'Email open rates vary by time of day' (states the obvious, no stakes)
  • Strong hook: 'By changing when we send emails, we could reach 45,000 more customers daily' (quantifies the opportunity)
Data Science & Analytics | Skills Union

From Insight to Action

  • Every insight should connect to a decision: what does the audience need to do differently?
  • One main message per story — three to five supporting points maximum
  • State the call to action explicitly: What (specific action), When (timeline), How (steps)
  • Quantify the impact of inaction as well as action — 'if we do nothing, X happens'
  • Weak ending: 'Feature X has low adoption rates' (vague, no path forward)
  • Strong ending: 'Users who discover Feature X use it 3× more — we can increase adoption by 300% with one UX change' (clear insight + specific solution)

Structuring a Data Presentation

Executive summary, details, appendix

Data Science & Analytics | Skills Union

The Pyramid Principle for Data Presentations

  • Lead with the conclusion — busy stakeholders should see the answer on slide 1
  • Executive summary (top): one-sentence finding, one-sentence recommendation
  • Main body (middle): three to five supporting points with evidence charts
  • Appendix (bottom): detailed data tables, methodology, raw analysis for those who want to dig
  • Most audiences only read the top — make the top enough to act on
  • The appendix signals rigour without cluttering the main narrative
Data Science & Analytics | Skills Union

Slide-by-Slide Structure

  • Slide 1 — Title: topic and the one key message
  • Slide 2 — Context: background the audience needs (situation)
  • Slides 3–5 — Evidence: one chart per slide, each supporting the main message
  • Slide 6 — Insight: the main finding stated plainly (the 'so what')
  • Slide 7 — Recommendation: specific action with timeline and owner
  • Appendix — Detail: methodology, full data tables, sensitivity analyses

Adapting for Your Audience

Technical vs business stakeholders

Data Science & Analytics | Skills Union

Know Your Audience

  • What do they already know? — calibrate complexity to their technical and domain expertise
  • What do they care about? — connect insights to their KPIs, goals, or pain points
  • What decisions do they need to make? — focus only on data that supports those decisions
  • Business stakeholders: lead with business impact, defer methodology to the appendix
  • Technical stakeholders: can handle model details, confidence intervals, and caveats in the main body
  • When in doubt, lead with business language and offer technical depth on request

Case Study 1

Walmart Sales Dashboard — before and after

Data Science & Analytics | Skills Union

Case Study 1: Walmart Sales Dashboard

  • Challenge: communicate monthly performance to 4,700+ store managers — original dashboard had 50+ KPIs
  • Bad version problems: too many metrics, inconsistent colours, no hierarchy, no context, scattered layout
  • Good version fix: reduced to 5–7 critical KPIs; green/red/neutral colour semantics; clear hierarchy by size and position; year-over-year comparisons; related metrics grouped together
  • Key lesson: store managers made better decisions with fewer, more relevant metrics
  • Key lesson: semantic colour (green = positive, red = negative) enabled instant scanning
  • Key lesson: most important metrics (daily sales, customer count) placed top-left where the eye lands first

Case Study 2

Spotify User Onboarding — making the journey visible

Data Science & Analytics | Skills Union

Case Study 2: Spotify User Onboarding

  • Challenge: 40% drop-off between signup and first playlist creation — original analysis was text-heavy paragraphs
  • Bad version problems: no flow diagram, missing drop-off metrics, no visual progression, no recommendations
  • Good version fix: funnel diagram showing each step with conversion rates; colour-coded stages (green/yellow/red); callout annotation at the worst step; specific recommendations per stage
  • Key lesson: visual journey maps were 3× more effective at communicating insights than prose
  • Key lesson: colour-coding by severity (green > 80%, yellow 50–80%, red < 50%) made bottlenecks instantly visible
  • Key lesson: tying each insight directly to a recommendation made the analysis actionable

Common Mistakes

Chart junk, misleading scales, and burying the insight

Data Science & Analytics | Skills Union

The Most Damaging Data Story Mistakes

  • Chart junk: decorative 3D effects, gradient fills, unnecessary borders — all obscure the data
  • Misleading scales: truncated y-axis or inconsistent intervals exaggerate or minimise differences
  • Data overload: showing every metric available forces the audience to decide what matters — your job is to do that for them
  • Burying the insight: putting the key finding on slide 12 after 11 slides of context — lead with it
  • No clear action: presenting analysis without a recommendation leaves the audience asking 'so what?'
  • Ignoring the audience: using technical jargon with business stakeholders (or over-simplifying for analysts)
Data Science & Analytics | Skills Union

Pre-Presentation Checklist

  • Is the key message visible on slide 1 without reading anything else?
  • Does every chart have an insight-driven title (not just a description)?
  • Is there one message per visualisation — no chart doing double duty?
  • Are colours used consistently and with semantic meaning throughout?
  • Have all unnecessary chart elements (gridlines, borders, legends) been removed?
  • Does the presentation end with a specific, measurable call to action?
  • Has a colleague reviewed it and can state the main message back to you in one sentence?
