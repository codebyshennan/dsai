Why Visualisation Matters

  • Speed: The brain processes visual information 60,000× faster than text
  • Memory: We retain ~80% of what we see vs ~20% of what we read
  • Pattern recognition: Visual patterns are far easier to spot than numerical ones
  • Decision-making: Clear charts lead to faster, better decisions
  • Pre-attentive processing: Colour, size, and position are detected in < 250 ms — before you even focus
  • Attentive processing: Interpreting meaning and relationships takes focused attention
  • Design tip: if the insight you want readers to notice is not pre-attentive, reinforce it with colour or size
Data Science & Analytics | Skills Union

Pre-attentive Attributes

  • Length / size: Bar heights, bubble areas — detected instantly without counting
  • Colour hue: Distinguishes categories at a glance
  • Colour intensity: Encodes ordered magnitude (heat maps, density plots)
  • Shape / marker: Differentiates series without relying on colour alone
  • Position: The most accurate channel for continuous values
  • Enclosure / grouping: Items inside a boundary are perceived as related
  • Use pre-attentive attributes to highlight the one thing you want viewers to notice first

Core Principles

Clarity, accuracy, efficiency, aesthetics

Core Principles of Good Visualisation

  • Clarity: One chart should answer one question — remove everything that does not help
  • Accuracy: Use zero-based axes for bar charts; never distort proportions or scales
  • Efficiency: Maximise the data-to-ink ratio — reduce chart junk and unnecessary decoration
  • Aesthetics: Clean typography, consistent spacing, and intentional colour build trust
  • Common pitfalls: truncated axes, 3-D effects, rainbow colour maps, missing labels or units
  • Gestalt rules: proximity groups related marks, similarity links same-type marks, continuity guides the eye along a path

Chart Type Selection

Match the chart to the question

Data Science & AI | Skills Union × Localized

Comparison — Which is largest?

  • Bar chart: Compare a few categories — sort by value, not alphabetically
  • Lollipop chart: Many categories where bar fill would create visual noise
  • Dot plot: Ranked items with two endpoints (before/after)
  • Small multiples: Compare the same chart across many subgroups
  • Rule: Sort bars by the metric being compared so rankings are immediately readable
Data Science & Analytics | Skills Union

Distribution — How is it spread?

  • Histogram: Distribution of a single continuous variable; choose bin width carefully
  • Density plot (KDE): Smooth alternative to a histogram; good for overlapping groups
  • Box plot: Shows median, IQR, whiskers, and outliers — compact multi-group comparison
  • Violin plot: Combines box plot with density shape for richer distribution view
  • Strip / swarm plot: Show every data point when n is small enough
Data Science & Analytics | Skills Union

Relationship, Composition & Trend

  • Scatter plot: Relationship between two continuous variables; add colour for a third
  • Bubble chart: Three variables at once — x, y, and bubble size
  • Heatmap / correlation matrix: Many pairwise relationships in one view
  • Stacked / grouped bar: Part-to-whole composition across categories
  • Line chart: Trend over an ordered dimension (time, distance, sequence)
  • Avoid pie charts for more than 3–4 segments — bar charts are almost always clearer

Colour Theory

Sequential, diverging, qualitative, and accessibility

Data Science & AI | Skills Union × Localized

Colour Palettes and Accessibility

  • Sequential: Ordered data — light-to-dark single hue (e.g. Blues, YlOrRd)
  • Diverging: Data with a meaningful midpoint — two hues diverge from neutral (e.g. RdBu, PRGn)
  • Qualitative: Unordered categories — distinct hues at equal brightness (e.g. Set1, tab10)
  • Accessibility: ~8% of men have colour-vision deficiency — use colorblind-safe palettes (viridis, cividis, Okabe-Ito)
  • Contrast: Maintain a minimum contrast ratio of 4.5:1 for text labels on chart backgrounds
  • Never encode meaning with colour alone — add shape, texture, or direct labels as a backup

Matplotlib Architecture

Figure, Axes, and the Artist model

Data Science & AI | Skills Union × Localized

Matplotlib Architecture

  • Figure: The top-level canvas that contains everything — set with plt.figure(figsize=(w, h))
  • Axes: The drawing area where data is plotted; one Figure can hold many Axes (subplots)
  • Artist model: Every visible element (line, patch, text, tick) is an Artist object attached to an Axes
  • pyplot interface: Stateful shorthand (plt.plot, plt.show) — quick for scripts and notebooks
  • Object-oriented interface: fig, ax = plt.subplots() — preferred for multi-panel figures and reusable functions
  • Key rule: do all drawing on ax, not on plt, when you have multiple subplots

Common Chart Types in Matplotlib

Line, bar, scatter, histogram, box

Data Science & AI | Skills Union × Localized

Core Matplotlib Chart Types

  • Line: ax.plot(x, y) — trends over time; use linewidth, linestyle, label
  • Bar: ax.bar(categories, values) — comparisons; use ax.barh for horizontal layout
  • Scatter: ax.scatter(x, y, c=colours, s=sizes) — relationships; supports colour and size channels
  • Histogram: ax.hist(data, bins=30) — distributions; choose bins carefully
  • Box plot: ax.boxplot(data) or ax.boxplot([g1, g2]) — side-by-side group distributions
  • Saving: fig.savefig('output.png', dpi=300, bbox_inches='tight') for publication-quality exports

Data Preparation for Visualisation

Aggregation, reshaping, normalisation

Data Science & AI | Skills Union × Localized

Data Prep: The Pre-Flight Checklist

  • Start with the question: State it in one sentence — it determines the data shape you need
  • Filter first: Remove rows outside scope (wrong period, cancelled status) before summarising
  • Aggregate to the right level: One row per visual mark — use groupby().agg() for totals or averages
  • Sort for readability: Sort bars by the metric, not alphabetically, so rankings are scannable
  • Reshape when needed: melt() converts wide tables to long form for Seaborn hue and Plotly Express
  • Handle missing values: Drop, impute, or show gaps — never silently treat NaN as zero

Annotations and Highlighting

Callouts, reference lines, selective emphasis

Data Science & AI | Skills Union × Localized

Directing the Viewer's Eye

  • Annotate sparingly: Only the maximum, a sudden change, a benchmark, or an outlier
  • Direct labels: ax.annotate() with arrowprops — eliminates legend round-trips
  • Reference lines: ax.axhline() / ax.axvline() for targets, averages, or event dates
  • Shaded ranges: ax.axhspan() / ax.axvspan() for acceptable bands or time windows
  • Selective colour: Highlight one bar in accent colour and grey the rest — strongest emphasis technique
  • Annotation text rule: explain *why* the value matters, not just what the value is

Troubleshooting Common Issues

Data Science & AI | Skills Union × Localized

Troubleshooting Guide

  • Overlapping labels: Rotate ticks with ax.tick_params(axis='x', rotation=45) or switch to horizontal bars
  • Blurry saved figures: Use dpi=150 or higher; use bbox_inches='tight' to avoid clipping
  • Memory leak in loops: Always call plt.close(fig) after saving to release figure objects
  • Wrong data shape: Check df.shape and df.dtypes before plotting; use groupby not raw rows
  • Colours not distinguishable: Switch to a perceptually uniform palette (viridis, plasma) or add markers
  • Annotations covering data: Use xytext offset and arrowprops to position text away from marks

Key Takeaways

  • Start with the question — chart type, data shape, and annotations all follow from it
  • Match the chart to the question: comparison → bar, distribution → histogram/box, relationship → scatter, trend → line
  • Use pre-attentive attributes (colour, size, position) to guide attention to what matters most
  • Prepare data before plotting: filter, aggregate, sort, reshape
  • Annotate with restraint — one highlighted element communicates more than ten
  • Make charts accessible: colorblind-safe palettes, sufficient contrast, direct labels over legends
