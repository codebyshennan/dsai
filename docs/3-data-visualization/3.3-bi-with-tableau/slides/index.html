Why Business Intelligence Exists

The chain that drives action

  • Data is everywhere: sales records, web logs, customer surveys, sensor readings
  • Insight is rare: most data sits in spreadsheets no one reads or databases no one queries
  • Decision is the goal: BI tools close the gap between raw numbers and the action a manager can take today
  • BI is not about building prettier charts — it is about making the right information visible to the right person at the right time
  • Example: a regional manager does not need a Python notebook; they need a dashboard that answers 'which territory is underperforming this quarter?' in one click

Three BI Tools Compared

Tableau vs Power BI vs Looker Studio

  • Tableau — Cost: paid (Public is free) | Platform: Windows + Mac | Best for: visual exploration, advanced analytics, mixed environments
  • Power BI — Cost: free desktop, paid cloud | Platform: Windows (Mac via browser) | Best for: Microsoft / Excel shops, enterprise reporting with DAX
  • Looker Studio — Cost: fully free | Platform: web browser, any OS | Best for: Google Analytics, Google Sheets, quick sharing
  • All three support live connections, scheduled data refresh, and embedded dashboards
  • Google-first org? → Looker Studio. Microsoft-first? → Power BI. Mixed or advanced analytics? → Tableau
  • For this course, lessons focus on Tableau with companion guides for Looker Studio and Power BI

Key Vocabulary

Five terms you will see constantly

Data Science & AI | Skills Union × Localized

Dimension

A categorical field — something you group or filter by

  • Examples: Region, Product Category, Customer Name, Order Date
  • Dimensions are shown in blue in the Tableau field list
  • When you drag a dimension to a shelf, Tableau creates a separate mark for each distinct value
  • Think of dimensions as the who / what / where / when of your data
Data Science & Analytics | Skills Union

Measure

A numeric field you aggregate — sum, average, count

  • Examples: Sales, Profit, Quantity, Discount
  • Measures are shown in green in the Tableau field list
  • When you drag a measure to a shelf, Tableau aggregates it (SUM by default)
  • Think of measures as the how much / how many of your data
Data Science & Analytics | Skills Union

Dashboard, Filter, and Data Source

The three connective tissue terms

  • Dashboard: a canvas holding multiple charts that update together — e.g. KPI tiles + map + trend line on one screen
  • Filter: a control that limits which rows appear in a view — e.g. show only orders from 2023 or only the West region
  • Data Source: the connection between Tableau and your data — an Excel file, a PostgreSQL table, a Google Sheet, a cloud warehouse
  • Live connection: Tableau queries the source on every interaction — always current, slower on large tables
  • Extract: a cached snapshot stored locally — faster offline, point-in-time, needs scheduled refresh

Four Steps from Data to Dashboard

The Tableau workflow

  • 1. Connect: point Tableau at a source — CSV, Excel, SQL database, Google Sheets, or a cloud warehouse. Set data types and join tables in the Data Source pane.
  • 2. Build: drag dimensions and measures onto the Rows and Columns shelves. Use the Marks card (Color, Size, Label, Tooltip) to add visual encoding. 'Show Me' suggests chart types.
  • 3. Analyze: add filters, reference lines, trend lines, and calculated fields. Use the Analytics pane for forecasting, clustering, and box plots — no formula required.
  • 4. Share: combine worksheets into a Dashboard, add filter actions so clicking one chart updates others, then publish to Tableau Public, Server, or Cloud.

Your First Chart in Tableau

Drag-and-drop in four steps

Building Your First Bar Chart

Using Sample Superstore (ships with Tableau)

  • Step 1 — Connect: open Tableau → start screen → 'More…' under Saved Data Sources → pick Superstore. You now see a field list on the left: blue dimensions, green measures.
  • Step 2 — Drag fields: drag Category (dimension) to the Columns shelf. Drag Sales (measure) to the Rows shelf. Tableau draws a bar chart and aggregates sales by category automatically.
  • Step 3 — Add colour: drag Region to the Color mark on the Marks card. Bars split into four colored segments — one per region. A legend appears on the right.
  • Step 4 — Add a filter: drag Order Date to the Filters shelf → select 'Years' → pick a year. The chart updates instantly to show only that year's data. You just built an interactive chart.

Design for the person who will use it, not the person who built it

Four Rules for Effective Dashboards

Principles that apply in any BI tool

  • One question, one dashboard: every element on screen should help answer a single business question. If you need two questions answered, use two tabs.
  • Eye path first: place the most important number top-left (LTR readers start there). KPI tiles across the top, main chart in the centre, supporting detail below.
  • Interactivity reduces clicks: a filter action (click a region on the map → all charts update) replaces five manual filter changes. Every interaction should save effort, not add it.
  • Test on the actual audience: show the dashboard to one stakeholder before publishing. If they cannot find the answer in 30 seconds, redesign — not relabel.

Power BI Workflow at a Glance

Same goal as Tableau, different vocabulary

  • Get Data: connect to Excel, SQL Server, Azure, SharePoint, or 100+ other sources via the Home ribbon → Get Data
  • Power Query: the data preparation layer — rename columns, filter rows, unpivot tables, merge queries. Changes are recorded as reproducible steps.
  • Data Model: define relationships between tables in the Model view. Power BI uses a star schema — one fact table surrounded by dimension tables.
  • DAX measures: Data Analysis Expressions is Power BI's formula language. Example: Profit Margin = DIVIDE(SUM(Sales[Profit]), SUM(Sales[Sales]))
  • Report canvas: drag visuals from the Visualizations pane, bind fields to visual properties (Axis, Values, Legend), and add slicers for filtering
  • Publish to Power BI Service: click Publish → dashboard is live in the browser; schedule automatic data refresh from the cloud

Looker Studio Essentials

Google's free, browser-based BI tool

Data Science & AI | Skills Union × Localized

Looker Studio Workflow at a Glance

Fastest path from Google data to shareable report

  • Connect a data source: click 'Create → Data Source' → choose a connector (Google Sheets, BigQuery, Google Analytics, CSV upload, 800+ partner connectors)
  • Create a report: click 'Create → Report' → add the data source. A blank canvas appears — no install, no desktop app.
  • Add charts: click 'Add a chart' in the toolbar → pick a chart type → drag fields to Dimension and Metric slots in the right panel
  • Add controls: date range control and filter dropdown are the two most useful interactive elements — drag them from the toolbar
  • Share: click the Share button → send a view-only link or embed via iframe. Anyone with a Google account (or the link) can view it without installing anything.
  • Best for: Google Analytics traffic reports, Google Ads performance, Google Sheets data, or any team already living in Google Workspace
