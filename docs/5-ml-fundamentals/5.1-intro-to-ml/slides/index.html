What is Machine Learning?

  • Traditional Programming: Rules + Data → Output (you write the logic explicitly)
  • Machine Learning: Data + Output → Rules learned automatically
  • ML lets computers find patterns from examples instead of following hand-crafted instructions
  • Example: Instead of writing spam filters with keyword rules, we show 10,000 labelled emails and the model learns the rules itself
  • When to use ML: when the rules are too complex to write by hand, or when patterns change over time
  • When NOT to use ML: when a simple rule or query already works, or when you need a fully auditable decision path

Types of Learning

Supervised, unsupervised, and reinforcement

Data Science & Analytics | Skills Union

Supervised Learning

  • Definition: Learn from labelled examples — each input has a known correct output
  • Regression: Predict a continuous number
  • → House prices, demand forecasting, temperature prediction
  • Classification: Predict a category
  • → Spam detection, image recognition, churn prediction
  • Requires a labelled dataset; performance is measurable against held-out ground truth
Data Science & Analytics | Skills Union

Unsupervised & Reinforcement Learning

  • Unsupervised learning: No labels — the algorithm finds structure on its own
  • → Customer segmentation (clustering), anomaly detection, topic modelling
  • → Dimensionality reduction: compress many features into 2–3 for visualisation (PCA, t-SNE)
  • Reinforcement learning: An agent learns by interacting with an environment and receiving rewards or penalties
  • → Game-playing AI (AlphaGo), robot navigation, resource scheduling
  • Most production ML today is supervised; unsupervised is common for EDA and pre-processing

Algorithm Families Overview

Regression, classification, clustering, dimensionality reduction

Data Science & Analytics | Skills Union

Common Algorithm Families

  • Linear models: Linear/logistic regression — fast, interpretable, good baseline
  • Tree-based models: Decision trees, Random Forest, Gradient Boosting — handle non-linearity, mixed types
  • Support Vector Machines: Effective in high-dimensional spaces; less common for tabular data now
  • Clustering: k-Means, DBSCAN — group similar observations without labels
  • Dimensionality reduction: PCA, t-SNE, UMAP — compress features for visualisation or denoising
  • Rule of thumb: always start with a simple linear model as your baseline before trying complex ones

The ML Workflow

Problem → Data → Features → Model → Evaluation → Deployment

Data Science & Analytics | Skills Union

ML Workflow: Problem Definition & Data

  • Step 1 — Problem definition: What question are we answering? What is success? (metric, threshold, business impact)
  • Step 2 — Data collection: Identify sources; check volume, coverage, and legal/privacy constraints
  • Step 3 — EDA & cleaning: Inspect shape, dtypes, missing values, outliers, and class balance
  • Key question before any modelling: 'Is this a regression, classification, or clustering problem?'
  • A poorly defined problem is the most common reason ML projects fail
Data Science & Analytics | Skills Union

ML Workflow: Modelling & Evaluation

  • Step 4 — Feature engineering: Transform raw columns into inputs the algorithm can use
  • Step 5 — Model selection & training: Start simple; compare multiple algorithms on a validation split
  • Step 6 — Evaluation: Report metrics on the held-out test set only (MAE, RMSE, accuracy, F1, etc.)
  • Step 7 — Deployment & monitoring: Save model artefacts; monitor for data drift and performance degradation
  • Data splits: training (fit) → validation (tune) → test (final, unbiased estimate)
  • Never peek at the test set during development

Feature Engineering

Selection, extraction, and transformation

Data Science & Analytics | Skills Union

Feature Engineering

  • What it is: Turning raw columns into inputs that algorithms can use effectively
  • Feature types: Numerical (continuous, discrete), categorical (nominal, ordinal), temporal, text
  • Scaling / normalisation: StandardScaler (z-score) or MinMaxScaler — required by distance-based and linear models
  • Encoding categoricals: One-hot encoding for nominal; ordinal encoding for ordered; avoid label encoding for nominal data
  • Derived features: Domain-informed combinations (price per sqft, is_renovated, age × condition)
  • Feature selection: Remove redundant or noisy inputs to reduce variance (SelectKBest, Lasso, correlation filter)

Bias–Variance Tradeoff

Underfitting, overfitting, and the sweet spot

Data Science & Analytics | Skills Union

Bias–Variance Tradeoff

  • High Bias (Underfitting): Model too simple — misses real patterns; both train and CV scores are low
  • → Fix: more features, higher-capacity model, less regularisation
  • High Variance (Overfitting): Model too complex — memorises noise; train score is high but CV score is much lower
  • → Fix: more training data, simpler model, regularisation (Ridge/Lasso), feature selection
  • Good fit: Both scores are reasonably high and the gap between them is small
  • Analogy: teaching cat recognition — too few examples → misses breeds (bias); too many random examples → calls dogs cats (variance)
Data Science & Analytics | Skills Union

Reading Learning Curves

  • Training score: How well the model performs on data it has seen
  • Cross-validation score: How well it performs on unseen data — the number that matters
  • Large persistent gap (train high, CV low): Overfitting — simplify or regularise
  • Both curves low: Underfitting — add complexity or engineer better features
  • Curves converge near a high value: Good fit — stop tuning, move to final evaluation
  • Learning curves are the single best diagnostic tool before hyperparameter tuning

Practical Tips

Baselines, data splits, and reproducibility

Data Science & Analytics | Skills Union

Practical Tips for ML Projects

  • Always establish a baseline: A simple mean predictor or majority-class classifier sets the floor
  • Set a random seed: `random_state=42` in every split and model for reproducible results
  • Split before scaling: Fit scalers and encoders on training data only; transform validation/test with the same fitted object
  • Version control everything: Code, data hashes, model artefacts, and performance metrics
  • Document assumptions: Record which rows were excluded, what features were engineered, and why
  • Monitor after deployment: Watch for data drift — the real world changes and models degrade silently

Key Takeaways

  • ML learns rules from data rather than following hand-coded instructions
  • Three learning paradigms: supervised (labels), unsupervised (no labels), reinforcement (rewards)
  • Algorithm families: linear models, tree-based, clustering, dimensionality reduction — start simple
  • Follow the workflow: problem → data → features → model → evaluation → deployment
  • Feature engineering often matters more than algorithm choice — invest time here
  • Diagnose with bias–variance: underfitting → add complexity; overfitting → simplify or regularise
